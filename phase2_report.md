# AutoDevOps: AIによる自動バグ修正パイプライン (Phase 2) 

このドキュメントでは、Phase 2 で構築した「AIがバグを検知し、修正コードを提案して、自動でGitHubのPull Request（PR）を作成するシステム」の仕組みと実行方法を誰にでもわかるように解説します。

---

## 🌟 概要：何ができるのか？

システム（`pipeline.py`）を実行すると、以下の処理が**すべて自動**で行われます。

1. **エラー情報の取得**: 本番環境で起きたエラーと、その原因と思われるソースコードを読み込みます（現在はテスト用のダミーファイルを使用）。
2. **AIによる解析と修正**: 最新のAI（Gemini 2.5 Flash）がエラー原因を特定し、コードを修正します。
3. **データベースへの保存**: 「どんなエラーだったか」「どう修正したか」という記録を Firebase (Firestore) に保存します。
4. **GitHubへの反映**: 対象のGitHubリポジトリに新しいブランチを作成し、修正済みのコードをコミット（保存）して、エンジニアにレビューを依頼する Pull Request (PR) を自動作成します。

---

## 🛠️ 事前準備（初回のみ）

このシステムを動かすには、いくつかの設定が必要です。

### 1. 必要なファイルを用意する
`scripts/` フォルダの中に以下のファイルがあることを確認してください。
*   `scripts/.env` : Google Cloud や GitHub の設定情報が書かれたファイル（`.env.example` をコピーして作成します）。
*   `scripts/secrets/private-key.pem` : GitHub を操作するための秘密鍵ファイル（セキュリティ上、絶対にGitHubに公開してはいけません）。

### 2. Google Cloud の認証を行う
ターミナルを開き、以下のコマンドを実行して Google アカウントでログインします。これにより、AI（Gemini）やデータベース（Firestore）にアクセスできるようになります。
```bash
gcloud auth application-default login
```

### 3. 必要なパッケージをインストールする
Python の実行に必要なツール（ライブラリ）をインストールします。
```bash
cd scripts
pip install -r requirements.txt
```

---

## 🚀 実行方法

準備ができたら、ターミナルで以下のコマンドを実行するだけです。

```bash
# scripts フォルダに移動（すでに移動している場合は不要）
cd scripts

# パイプラインを実行
python pipeline.py
```

### 実行中の画面表示（イメージ）
実行すると、ターミナルに以下のような進行状況が表示されます。

1. `🔌 初期化中...`
2. `📥 GitHubからファイルを取得中...`
3. `🤖 Gemini (gemini-2.5-flash) に解析を依頼中...` （※AIが考えているので数秒〜数十秒かかります）
4. `✅ 解析完了: エラー原因と修正内容の説明...`
5. `💾 Firestoreに保存中...`
6. `🚀 Pull Requestを作成中...`
7. `🎉 パイプライン実行完了！`

---

## ✅ 確認方法（実行が成功したら）

ターミナルの最後に表示される2つのリンクを確認してください。

1. **GitHub の確認**
   *   表示された GitHub の URL をクリックします。
   *   自動で作成された Pull Request（PR）の画面が開き、AIが修正したコードの「変更点（Diff）」が確認できます。
2. **Firebase (Firestore) の確認**
   *   Firebase Console を開き、Firestore Database の `auto_fixes` コレクションを確認します。
   *   エラー内容や修正後コード、作成された PR の URL がデータベースに記録されていることがわかります。

---

## 💡 よくあるエラーと対処法

*   **エラー**: `DefaultCredentialsError` や `403 Cloud Firestore API has not been used`
    *   **原因**: Google Cloud の認証が切れているか、プロジェクト設定が間違っています。
    *   **対処法**: `gcloud auth application-default login` を再実行し、正しいアカウントを選択してください。
*   **エラー**: `FileNotFoundError: secrets/private-key.pem`
    *   **原因**: GitHub App の秘密鍵ファイルが所定の場所にありません。
    *   **対処法**: `scripts/secrets/` フォルダを作成し、正しい `.pem` ファイルを配置してください。
*   **エラー**: `JSONDecodeError` または `Gemini の応答を解析できませんでした`
    *   **原因**: AIが想定外のフォーマットで回答を返してきた場合のエラーです（プロンプトの調整により現在はほぼ発生しません）。
    *   **対処法**: もう一度 `python pipeline.py` を実行してみてください。AIの出力が安定すれば成功します。
